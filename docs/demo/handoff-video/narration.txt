What happens when one agent has the context, but another agent needs to finish the work?

TriSeek records the useful parts of the session: searches, file freshness checks, pinned snippets, and the current working set.

Here the work starts in a Claude session. We open a portability session, record useful context, and create a snapshot.

The snapshot is just local files. It can also carry a compact briefing for the next harness.

Now Codex resumes from the same snapshot. The daemon is hydrated, and a resume payload is written for the new session.

The handoff is not magic. It is a reproducible bridge: snapshot, brief, resume, and verify.
