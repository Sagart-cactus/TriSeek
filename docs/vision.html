roadmap

Local context substrate

TriSeek is becoming the local context substrate that agent harnesses share. Today that means portability between Claude Code and Codex on the same machine. Tomorrow it means more harness adapters, cross-machine resume, audit, and replay without giving up local-first or bounded output.

What's shipped

  • Portable work sessions with `session_*` MCP tools.
  • Snapshot, resume, pack export/import, and no-inference briefings.
  • First-class Claude Code ↔ Codex adapter flows on the same machine.

What's next

Planned First-class portability adapters for OpenCode and Pi.
Designing Eval harness for snapshot quality and dead-end re-exploration rate.
Planned Replay and audit verbs.
Planned Cross-machine resume through user-controlled pack sync.
Designing Auto-snapshot via harness hooks.
Considering Cursor, Cline, and Aider adapters.
Considering Symbol graph and repo map.

Consciously not building

  • A full agent-first browser.
  • An identity or permissions layer.
  • A sandbox runtime.
  • Embeddings or semantic search yet.
  • An action registry outside TriSeek's search and context discipline.

Commitments

  • Local-first by default.
  • Bounded MCP output, always.
  • Daemon performs no inference.
  • Repo-scoped trust model.
  • Open source under MIT.
