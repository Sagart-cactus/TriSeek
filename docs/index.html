One substrate, many context jobs

TriSeek is not only a faster grep and not only a handoff helper. It is a local product surface that gives agents the context primitives they need before, during, and after a coding session.

01 / FIND

Code search

Trigram-indexed content and path search with ripgrep fallback, backend receipts, and bounded MCP envelopes.

02 / REMEMBER

Memo & reuse

Session-aware freshness checks prevent stale skips and let agents reuse unchanged reads and duplicate search results.

03 / ORIENT

Context packs

Small ranked starting sets for bugfix and review work: files, clipped snippets, reason tags, and next-step hints.

04 / CARRY

Sessions

Working sets, action logs, pinned snippets, git state, frecency, and snapshots live outside the repo.

05 / RESUME

Handoff

Claude Code and Codex can exchange a hydrated session today; other harnesses can use the same primitives manually.

Built for the way agent sessions actually unfold

The useful layer is not one giant context dump. It is a set of bounded, local operations that help the agent start with the right files, avoid redundant work, and recover from harness changes.

From cold task to warm continuation

  • 1Start with a pack. Give the agent a compact, ranked set of likely files instead of asking it to scan the world.
  • 2Search with receipts. Every response says which backend answered and clips output to fit MCP budgets.
  • 3Reuse context honestly. Memo and search reuse only skip output while files are still fresh for the session.
  • 4Snapshot before losing state. Capture the working set, action log, git state, and frecency when a session needs to move.

For agent-tool builders

TriSeek is a neutral local layer that exposes stable MCP tools and CLI commands without requiring a cloud account, identity layer, or model dependency. The daemon does search, context accounting, and session state; the harness keeps execution and editing authority.

The shipped feature surface

Each feature is useful on its own. Together they form the local context substrate that multiple agents and harnesses can share.

⚡
Fast local search
Pre-built trigram indexes speed repeated code queries, with direct scan or ripgrep fallback when that is the better path.
⬡
MCP native
Drop-in stdio MCP server for Claude Code, Codex, OpenCode, Pi, and other MCP-capable clients.
◇
Memo & search reuse
Tracks session reads and duplicate searches so agents can reuse context only while the underlying files remain fresh.
◎
Context packs
Intent-aware trailheads for bugfix and review tasks with ranked paths, snippets, reason tags, and hints. CLI and MCP.
⇄
Context handoff
Snapshot Claude Code and Codex work sessions with primed memo, warmed search reuse, restored frecency, and captured git state.
▣
Portable packs
Export and import .tcp snapshots between checkouts of the same repo without putting state inside the repository.
▦
Bounded by design
Clipped responses, deduped matches, preview truncation, and explicit budgets keep agents from drowning in tool output.
◯
Local-first default
Core daemon has no network calls, telemetry, OAuth, or vendor cloud. Briefings are explicit user-triggered outputs.
◉
Release validation
Docker validation covers CLI, MCP, daemon isolation, search reuse, context packs, and supported agent installers.
