Briefing Modes

A briefing is a markdown file that summarizes what the agent was trying to do and what TriSeek captured. It lives at briefing.md inside a snapshot directory and is optional.

The shipped modes

ModeWhat it does in v0.4.2When to usePrivacy
no-inferenceBuilds a fixed template from the action log.Default; fastest; no setup.100% local.
local-modelAccepted CLI mode, but currently falls back to the same no-inference template.Reserved for model-backed local summarization.No external calls in v0.4.2.
cloud-modelAccepted CLI mode, but currently falls back to the same no-inference template.Reserved for explicit cloud summarization.No external calls in v0.4.2.

Commands

triseek brief <snapshot_id> --mode no-inference
triseek brief <snapshot_id> --mode local-model
triseek brief <snapshot_id> --mode cloud-model
triseek handoff codex --from claude_code --brief-mode no-inference

There is no user config file for briefing providers in v0.4.2. The accepted model modes are placeholders that still generate the no-inference briefing.

The briefing schema

## Goal
## Current hypothesis
## Established facts
## Ruled out
## Open questions
## Next planned action

Ruled out is the highest-leverage section because it tells the next harness what not to re-explore.

Validation

TriSeek validates required headers, checks action-log citations against captured entries, and enforces a 6000-byte budget. Failed validation aborts the briefing write instead of leaving a malformed file.
