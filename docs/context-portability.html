SINCE v0.4.2 · CLAUDE CODE ↔ CODEX

Switch between Claude Code and Codex without losing the work.

Snapshot a session in one harness. Resume it in the other with primed memo, warmed search reuse, and restored frecency. Local-first by default. Bounded by design.

// vendor contingency

Claude Code goes down at 2pm. Your work doesn't have to.

Multi-harness reality

Serious AI-coding teams already pay for both Claude Code and Codex. Running both in parallel is normal.

Context loss is expensive

Four hours of bug exploration is meaningful work. Starting fresh in the other harness wastes most of it.

Neutral layer

No harness vendor has incentive to commoditize its own context. TriSeek sits between Claude Code and Codex.

// how it works

Five-step flow

1. Open

session_open declares the work session goal.

2. Work

TriSeek calls accrue to the action log automatically.

3. Snapshot

session_snapshot captures working set, action log, frecency, git state, and snippets.

4. Brief

triseek brief writes an optional markdown summary into the snapshot.

5. Resume

session_resume primes daemon state and emits a hydration payload.

// snapshot contents

What's in a snapshot

A snapshot contains manifest.json, working_set.json, action_log.jsonl, pinned_snippets.json, pinned_snippets/<sha>.txt, git_state.json, and optional briefing.md. See the Pack Format page for schemas.

The daemon performs no inference. Briefing is an explicit CLI command. Snapshots live under <daemon_dir>/snapshots/. Raw file contents are not copied, except user-selected pinned snippets.

// walkthroughs

Concrete walkthroughs

Claude Code → Codex

triseek snapshot create --session session_123 --source-harness claude_code .
triseek brief <snapshot_id> --mode no-inference
triseek resume <snapshot_id> --write-to AGENTS.md

Claude Code → Claude Code

triseek snapshot create --session session_123 --source-harness claude_code .
triseek resume <snapshot_id> --write-to CLAUDE.md
// limits

What v0.4.2 does not promise

Same-machine only, manual snapshot only, single-user packs, and first-class portability adapters for Claude Code and Codex only. See the Vision page for what comes next.
