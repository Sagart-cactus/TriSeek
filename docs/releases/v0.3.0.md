# TriSeek v0.3.0

TriSeek v0.3.0 turns Memo into a first-class shipped surface alongside local indexed search. This release adds agent-visible Memo tools, installs Memo hooks/plugins for supported clients, introduces active-mode `memo_check` for Codex, and adds an in-process query cache for repeated MCP search calls.

## Highlights

- Shipped Memo as a supported agent subproduct with daemon-backed session freshness state.
- Added Memo MCP tools:
  - `memo_status`
  - `memo_session`
  - `memo_check`
- Added passive Memo install flows for Claude Code, OpenCode, and Pi.
- Added active Memo guidance for Codex, including `memo_check` for the current upstream hook limitation.
- Added stale-file token delta hints via `current_tokens` on stale Memo responses.
- Added an in-process MCP search query cache with bounded LRU storage, TTL expiry, and reindex invalidation.
- Expanded regression coverage for Memo daemon logic, Memo MCP forwarding, installer template generation, and query-cache behavior.

macOS and Linux:

```sh

```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.ps1 | iex"
```

## Notes

- Memo is passive on Claude Code, OpenCode, and Pi, but active on Codex for now because Codex hooks still do not reliably fire for non-Bash tools.
- Search tools still run directly inside the MCP server; Memo state remains daemon-backed in this release.
- Search responses now expose `cache: "hit" | "miss" | "bypass"` so clients can reason about repeated-query behavior.
- Release validation for this branch includes workspace fmt/clippy/test, Memo daemon tests, MCP Memo forwarding tests, query-cache integration tests, installer template assertions, a release-binary build, and `scripts/release_smoke.sh`.
