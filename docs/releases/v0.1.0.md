# TriSeek v0.1.0

TriSeek's first tagged release packages the indexed search CLI, the background daemon, and the new one-command installation flow.

## Highlights

- Added a background daemon with local RPC control, idle shutdown, live index reloads, and daemon lifecycle commands through `triseek daemon`.
- Added filesystem watching and incremental index updates so a running daemon can keep the index fresh without full rebuilds on every change.
- Added frecency-based reranking and explicit selection tracking so repeated searches can learn from recent usage.
- Added release-based installers for macOS, Linux, and Windows, with Cargo fallback when a matching release archive is not available.
- Added packaged `triseek-server` binaries to CI and release artifacts so daemon support works after install.
- Added Windows support for the daemon control path by replacing the Unix-only socket transport with a cross-platform loopback TCP transport.

## Install

macOS and Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.sh | sh
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.ps1 | iex"
```

## Notes

- This release is intended for local indexed search in medium and large repositories.
- The daemon uses a local loopback transport and writes repo-local PID and port files under the index directory.
- After this release is published, the installer path will use prebuilt release assets instead of falling back to Cargo.
