# TriSeek v0.2.0

TriSeek v0.2.0 adds a first-class MCP server for Claude Code and Codex, keeps the existing CLI search behavior on the same shared routing pipeline, and hardens the MCP path with regression coverage for bounded output, no-index bootstrap, and repeated-session performance.

## Highlights

- Added a local stdio MCP server via `triseek mcp serve` with a small frozen tool surface: `find_files`, `search_content`, `search_path_and_content`, `index_status`, and `reindex`.
- Added one-command install and uninstall flows for Claude Code and Codex, plus `triseek doctor` for environment verification.
- Extracted the hybrid routing pipeline into a shared `search_runner` so CLI search and MCP tool calls follow the same indexed, direct-scan, and ripgrep fallback logic.
- Cached the opened search index across MCP requests within a session and invalidate it on reindex so repeated agent searches stay fast without serving stale results.
- Fixed MCP result limiting so a single file with many matches cannot exceed the requested `limit`.
- Fixed incremental `reindex` so it bootstraps correctly when no index exists yet.
- Corrected `index_status.indexed_files` to report repo-wide searchable files instead of only the last delta-update batch.
- Added end-to-end MCP server integration tests covering protocol handshake, tool execution, bounded regex output, missing-index bootstrap, and cached-engine invalidation.

## Install

macOS and Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.sh | sh
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.ps1 | iex"
```

## MCP Quickstart

```sh
triseek install claude-code --scope project
triseek install codex
triseek doctor
```

## Notes

- This release is intended for local indexed search in medium and large repositories, now with a supported MCP path for agent clients.
- MCP transport is stdio-only and local-only; TriSeek exposes search tools but no shell or file-editing tools.
- Existing installer flows still prefer published GitHub Release assets and fall back to Cargo when a matching asset is not available.
