# TriSeek v0.2.1

TriSeek v0.2.1 shifts the product toward a standalone local code-search CLI: search no longer requires the `search` keyword, index state is centralized under `~/.triseek`, and one global daemon can serve multiple roots without leaving repo-local index artifacts behind.

## Highlights

- Added bare-search CLI usage so `triseek "needle" [path]` works without `search`.
- Made path arguments positional across the main CLI flows, including `build`, `update`, `measure`, and daemon commands.
- Canonicalized `.`, `..`, relative paths, and symlinked roots so equivalent paths reuse the same index directory.
- Moved default index storage out of repo roots into `~/.triseek/indexes/<root-key>/`.
- Reworked daemon lifecycle around one global daemon with global control files under `~/.triseek/daemon/`.
- Kept repo and root data isolated inside per-root index directories, including frecency state.
- Updated install, doctor, and MCP-facing messaging to reflect the new standalone CLI UX and centralized storage layout.
- Added a deeper manual regression plan covering CLI, daemon, MCP, frecency, install, performance, and robustness flows.

## Install

macOS and Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.sh | sh
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://raw.githubusercontent.com/Sagart-cactus/TriSeek/main/scripts/install.ps1 | iex"
```

## Notes

- Default operation no longer writes index artifacts into searched repo roots.
- The MCP server remains repo-scoped per process for now, while the background search daemon is global.
- Local release validation on this branch included `cargo fmt --all --check`, `cargo clippy --workspace --all-targets --locked -- -D warnings`, `cargo test --workspace --locked`, and targeted CLI/daemon/MCP smoke checks.
