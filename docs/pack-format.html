Pack Format

A pack is a gzipped tarball that materializes a snapshot directory. Extension: .tcp. Schema version: 1. v1 is unsigned.

Layout inside the tarball

<root>/
  manifest.json
  working_set.json
  action_log.jsonl
  pinned_snippets.json
  pinned_snippets/
    <sha>.txt
  git_state.json
  briefing.md        # optional
  checksums.txt

Artifact schemas

ArtifactRequired fieldsDescription
manifest.jsonschema_version, snapshot_id, session_id, created_at, repo_root, generation, context_epochSnapshot identity, repo, git, harness, and daemon generation metadata.
working_set.jsonschema_version, files_read, searches_run, frecency_top_nThe state TriSeek can hydrate into a new harness session.
action_log.jsonlschema_version, entry_id, session_id, ts, kind, payloadAppend-only log of captured TriSeek calls.

What's deliberately not in the pack

No raw repo file contents, no chat transcripts, no API keys, no model parameters. Pinned snippets are the only verbatim content, and only for spans the prior agent intentionally pinned.

Forward compatibility

Schema version bumps on breaking reader changes. Additive fields do not bump the version. Readers validate the version and refuse unknown schemas.

Why this format

A directory of files stays debuggable, JSONL survives partial writes, content-addressed snippets dedupe naturally, and excluding raw file bodies keeps packs small and tied to the local repo checkout.
